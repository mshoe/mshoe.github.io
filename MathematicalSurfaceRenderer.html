Mathematical Surface Renderer

A program by Michael Xu

I wrote a program in C++ that renders mathematical surfaces using OpenGL. The algorithm renders surfaces through parametric equations. I wrote this algorithm myself because I couldn't find a good tutorial online for creating shaded-wireframe-mesh-looking surfaces. However, I'm pretty sure this algorithm is very similar to what every other mathematical renderer (Matlab, wolfram, etc.) uses, at least the part about parametric equations, because how else would you accurately determine which vertices are adjacent and figure out which vertices make up which quads?

I plan to use this software for an upcoming mobile application with my team Augmented Geometry. We got second place at WearHacks Toronto with an AR version of this software in Unity, even though we didn't know a thing about computer graphics. This new version is written in a much faster language for graphics-intensive software (C++) and uses a much better algorithm. Anyways, here are some pics:
